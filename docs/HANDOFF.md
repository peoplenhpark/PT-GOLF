# 운동 노트 앱 — 작업 핸드오프 문서

> **목적:** 이 문서 하나로 다른 에이전트가 현재까지의 맥락·결정·명세를 파악하고 곧바로 다음 단계(배포 가능한 PWA 구현)를 이어받을 수 있도록 정리한다.
> **최종 업데이트:** 2026-07-01 · **담당자:** Chris (chris@cutback.video)

---

## 0. TL;DR (다음 에이전트가 할 일)

확정된 설계대로 **설치형 PWA**를 실제 코드로 만들고 **GitHub Pages 배포**까지 구성한다.
- 다크·미니멀 톤, 2개 파트(PT / 골프), 동작 카드 누적 관리 + 큐 체크리스트 + 날짜별 메모 타임라인.
- **PT 파트는 "세션(일일수행)" 서브탭 구조**로 확장됨 — 레슨 세션이 늘어날 때마다 서브탭이 추가된다(§5, §6, §8).
- 데이터는 **로컬 우선(IndexedDB) + JSON export/import 백업**. 서버·로그인 없음.
- §6 시드 데이터(일일수행1 등 3종 + 일일수행2 하체 재활 4종)를 초기 데이터로 포함할 것.
- 인터랙티브 프로토타입은 이미 검증됨(§5). 그 UX/데이터 모델을 그대로 구현하면 된다.

## 1. 프로젝트 개요

Chris가 PT/골프 레슨에서 받은 코칭 노트를, **운동 중(헬스장·필드)에 한 손으로 쉽게 보고 메모할 수 있는 개인용 앱**으로 만들고 GitHub에 올리려 한다.

핵심 요구사항(사용자 원문 기준):
- **2개 파트**: 하나는 PT, 다른 하나는 골프.
- 동작(드릴)이 **부정기적으로 추가**된다 → 이를 **누적 관리**해야 함.
- 화면에 **디스플레이**하고, **메모**도 할 수 있어야 함.
- 잘 만들어진 유사 앱을 참고한 **앱 구성 제안**을 원함(완료, §7).

차별점: 일반 운동 트래커의 *세트/횟수 기록*이 아니라, **"나만의 코칭 노트(큐 + 메모)의 누적 관리"**에 무게를 둔다.

## 2. 진행 타임라인

| 단계 | 내용 | 상태 |
|---|---|---|
| 1 | 클로바노트 음성(PT 등 운동) 전사본을 사용자가 직접 붙여넣음 → 공백·중복 제거 후 "종합"을 중심축으로 요약 | ✅ 완료 |
| 2 | 요약을 바탕으로 **한 장짜리 PDF 체크리스트** 제작 (한글 폰트 렌더링 검증 완료) | ✅ 완료 |
| 3 | PDF 노트를 앱화하는 방향 논의 → 레퍼런스 조사 + 앱 구성 제안 | ✅ 완료 |
| 4 | 진행 방향 확정(아래 §4) → **다크·미니멀 인터랙티브 프로토타입** 제시 | ✅ 완료 |
| 5 | 2차 PT 세션(2026-07-01, 하체 재활) 전사본 요약 → **PT 서브탭 구조 도입, `일일수행2` 추가** + 동작 참고 일러스트 제작 | ✅ 완료 |
| 6 | **배포 가능한 실제 PWA 구현 + GitHub Pages 배포** | ⏳ 다음 단계 |

> 참고: 클로바노트 공유 URL(`clovanote.naver.com`)은 차단 도메인이라 직접 접근 불가. 전사 콘텐츠는 사용자가 채팅으로 제공했고, 그 내용이 §6 시드 데이터의 출처다.

## 5. 검증된 프로토타입 명세 (UX 기준선)

다크 폰 프레임(폭 390px) 안에서 동작하며 아래 인터랙션이 모두 검증됨:
- 상단 **PT / 골프 탭 전환** (파트별 액센트 색: PT teal, 골프 amber)
- **PT 서브탭(세션/일일수행)**: PT 선택 시 `일일수행1(06.08·등)`, `일일수행2(07.01·하체 재활)` … pill 형태로 나열, 세션 전환 시 해당 세션 동작만 표시. 기본은 최신 세션(`일일수행2`)이 열림. (골프는 서브탭 없음)
- **세션 강평 카드**: 각 세션 리스트 상단에 그 세션의 핵심 코칭 요약을 카드로 노출(§6의 세션별 summary)
- **검색** 바 (이름·태그 필터)
- **카드 리스트**: 이름, 태그 pill, 핵심 수치, 핀(즐겨찾기), 큐 진행도(n/총), 메모 개수. 핀 항목 상단 정렬
- 카드 탭 → **상세**: 태그, 핵심 수치, **큐 체크리스트(탭으로 토글, 완료 시 취소선)**, **메모 타임라인(날짜+텍스트)**, 하단 메모 입력 → 추가 시 오늘 날짜로 누적
- 우하단 **➕ FAB → 추가 폼**: 이름/태그/수치/큐(줄바꿈 구분)/첫 메모 입력 → 저장 시 리스트 맨 위에 추가
- 핀 토글, 백업 안내 문구 포함

이 화면 흐름과 데이터 모양을 그대로 구현 목표로 삼는다.

## 6. 시드 데이터 (초기 콘텐츠 — 반드시 포함)

> 출처: PT 레슨 전사본 요약. PT는 **세션(일일수행)** 단위로 묶는다. 골프는 구조 확인용 예시이므로 추후 실제 레슨 내용으로 교체 예정.

### PT · 일일수행1 (2026-06-08 · 등)

**1) 풀업 / 턱걸이** — 태그: `등`, `언더그립` · 수치: `60kg · 언더 위주` · 핀: true
- 큐:
  - 매달려 충분히 늘리기 → 엉덩이 살짝 쪼이기
  - 올라갈 때 「가슴이 먼저」 출발 (어깨 으쓱 X)
  - 턱·가슴이 하늘을 보게 열며 올라가기
  - 내려올 때 늘어난 느낌 그대로 당겨 올라가기
  - 몸 앞으로 굽히지 않기
- 메모(06.08): 언더 그립 위주로. 등·어깨 안정되며 등 늘리기 좋음. 가슴 먼저 출발 연습.

**2) 시티드 로우** — 태그: `등 상부`, `오버그립` · 수치: `20kg · 오버 위주`
- 큐:
  - 등을 감아 들어가듯 충분히 늘리기
  - 어깨 들리지 않게 명치 살짝 들며 일자로 당기기
  - 팔꿈치는 그립 결 방향 그대로 빠져나오게
  - 둘째 손가락을 은색에 걸기, 너무 넓게 X
- 메모(06.08): 오버 그립 위주 → 등 상부 발달. 상부 살면 하부 따라옴.

**3) 암풀다운 (스트레이트 암 풀다운)** — 태그: `등`, `보조` · 수치: `10kg × 15`
- 큐:
  - 빼서 잡기(꾹 X), 넓이 약 50cm
  - 힙힌지 — 배꼽보다 무릎이 무조건 뒤
  - 가슴이 바닥 보는 느낌 유지
  - 바가 턱쯤 오면 가슴 내밀며 당기기 (허리 꺾지 X)
  - 큰 원을 그리며 당겨오기
  - 세팅 자세 10개 내내 유지 / 겨드랑이 꽉 조이기
- 메모(06.08): 손바닥·손날로 감아쥐기. 등 운동 핵심 보조, 연습 많이.

**일일수행1 세션 강평(앱 상단 강평 카드로 노출):**
- 매 순간 「어느 근육이 늘어나고 어느 근육이 수축되는지」 인지하며 한다(고립).
- 무게 욕심 금지 — 가벼울수록 자기 힘으로 하는 것이라 더 힘들다. 어디를 쓰는지가 우선.
- 등 운동은 손날(새끼-약지 라인)·손바닥 힘으로 감아쥔다. 손가락 끝힘은 힘이 샌다.
- 「잡아채기」: 같은 자세라도 잡아채면 힘 들어가는 게 완전히 다르다 — 계속 연습.
- 자세 흐트러지면 멈춘다. 횟수보다 자세. 한 달만 제대로 하면 등에 힘 들어가는 게 느껴진다.

### PT · 일일수행2 (2026-07-01 · 하체 재활)

> 배경: 회원(어르신)이 **좌골 신경통** 진단이 있어, 하체는 무리한 근력 운동보다 **재활을 선행**한다. 아래는 실제 병원 재활 프로그램 기반의 기능성 운동으로, 모두 **좌우 각각** 수행한다.

**일일수행2 세션 강평(강평 카드):**
- 좌골 신경통 진단 → 하체 무리한 운동 지양, 재활 선행.
- 개수·무게보다 **완벽한 자세**가 우선 — 편한 자세로 도망가지 않기.
- 속근육(잔근육) 자극이라 전신 운동처럼 힘듦, 자세 무너지면 멈춤.
- 좌우 불균형: **오른쪽 = 기능적 문제(장요근 기능↓), 왼쪽 = 근력 문제** → 단측(일측) 운동으로 좌우 균형.
- 정석은 **20회 × 3세트**(오늘은 ~1세트). 매일 6개 다 말고 **생각나는 3~4개씩** 데일리로.
- 숙제 6가지: 오늘 4개(쿼드셋·SLR·클램쉘·SSLR) + **데드버그** + **폼롤러**. (브리지·이너타이는 안쪽 허벅지 통증 유발 가능성 있어 주의 / 플랭크는 허리 부담으로 트레이너와 함께.)

**1) 쿼드 셋 (Quad Set)** — 태그: `하체 재활`, `무릎 안정` · 수치: `10초 × 10회 (좌우)` · 핀: true

![쿼드 셋](images/01_quadset.png)

- 큐:
  - 무릎 아래 접힌 부위 살짝 아래에 수건 받치기
  - 앞꿈치(발꿈치) 당기고 엄지발가락 몸쪽으로
  - 허벅지 힘으로 수건을 눌러 10초 버티기
  - 천천히 내렸다가 끊기지 않게 반복
- 메모(07.01): 무릎 불안정하면 흔들림 정상. 오른쪽이 더 흔들려 → 수건 조금 높게. 단순하지만 허벅지 많이 써야.

**2) SLR (스트레이트 레그 레이즈)** — 태그: `하체 재활` · 수치: `15회 (좌우)`

![SLR](images/02_slr.png)

- 큐:
  - 골반 바닥 고정, 엉덩이 뜨지 않게
  - 앞꿈치 당기고 무릎 안 구부러지게 다리 쭉 펴서 들기
  - 많이 올릴 필요 없음 — 여기까지만
  - 내릴 때 뒤꿈치 바닥에 닿지 않게
- 메모(07.01): 10개 넘어가면 다리 달달 떨림 정상. 무릎 구부러지면 안 됨.

**3) 클램쉘 (Clamshell)** — 태그: `하체 재활`, `좌골` · 수치: `15~20회 (좌우)`

![클램쉘](images/03_clamshell.png)

- 큐:
  - 옆으로 누워 두 발 붙이고 무릎 세우기
  - 뒤꿈치는 딱 붙인 채(앞꿈치는 들려도 됨)
  - 무릎을 조개처럼 벌렸다 모으기
  - 밴드를 무릎에 걸면 자극·효과 ↑
- 메모(07.01): 옛 좌골 통증 지점을 정확히 자극. 단순하지만 효과 좋음.

**4) SSLR (사이드 스트레이트 레그 레이즈)** — 태그: `하체 재활`, `골반` · 수치: `20회 (좌우)`

![SSLR](images/04_sslr.png)

- 큐:
  - 옆으로 누워 몸을 완전히 일자로 펴기
  - 아래(앞) 다리는 살짝 앞으로 빼기
  - 위 다리 쭉 편 채 엉덩이·골반 힘으로 옆으로 들기
  - 손가락으로 장요근(골반 앞) 눌러주면 마사지 효과
- 메모(07.01): 엉덩이/골반 힘으로 든다고 생각. 기초 자세 유지가 핵심.

### 골프 (예시 — 교체 예정)

```
 ├─ [PT] 서브탭: 일일수행1 | 일일수행2 | …   (세션 전환, 기본=최신 세션)
 ├─ 리스트 화면
 │    ├─ 세션 강평 카드 (PT, 해당 세션 요약)
 │    ├─ 검색 + (태그/카테고리 필터)
 │    ├─ 동작 카드 그리드 (핀 우선, 최근 수정순)
 │    └─ ➕ 동작 추가 (FAB, 현재 세션에 추가)
 ├─ 상세 화면
 │    ├─ 태그 / 핀 / 핵심 수치
 │    ├─ 큐 체크리스트 (토글)
 │    ├─ 메모 타임라인 (날짜별 누적)
 │    └─ 메모 입력
 ├─ 추가·편집 폼
 └─ 설정 / 백업 (JSON export·import, PWA 설치 안내)
```

## 8. 데이터 모델

**계층 구조:** `파트(part) → [PT만] 세션(일일수행) → 동작(item)`. 골프는 세션 계층 없이 동작 배열을 바로 가진다.

세션(PT):

```json
{
  "id": "s2",
  "label": "일일수행2",
  "date": "2026-07-01",
  "focus": "하체 재활",
  "summary": ["세션 강평 bullet ...", "..."],   // 강평 카드에 표시
  "items": [ /* 동작 객체 배열 */ ]
}
```

동작(드릴) 1건:

```json
{
  "id": "q1",
  "name": "쿼드 셋 (Quad Set)",
  "category": "하체 재활",             // 선택
  "tags": ["하체 재활", "무릎 안정"],
  "metrics": { "display": "10초 × 10회 (좌우)" },  // 자유 텍스트 + 선택적 구조화 필드
  "cues": ["...", "..."],             // 체크리스트 항목
  "cueDone": [false, false],          // 큐별 체크 상태(개인 기록, 세션 리셋 옵션 고려)
  "notes": [ { "date": "2026-07-01", "text": "..." } ],  // 날짜별 누적
  "media": ["images/01_quadset.png"], // 이미지/영상 링크(선택) — 재활 동작은 참고 그림 연결
  "pinned": true,
  "createdAt": "2026-07-01T...",
  "updatedAt": "2026-07-01T..."
}
```

> 데이터 형태: `{ pt: { sessions: [세션…] }, golf: { items: [동작…] } }`. 새 PT 레슨 = 세션 1건 추가(= 서브탭 1개 추가). 세션 내 동작이 부정기적으로 추가되면 해당 `items`에 push.

저장 계층(2-tier, 확정안):
- **콘텐츠**(동작 정의·큐): 리포의 시드 JSON으로 출발 가능.
- **개인 기록**(체크 상태·메모): 브라우저 **IndexedDB**.
- **백업**: 전체 상태를 **JSON export/import** → 사용자가 GitHub에 커밋.

> 고려사항: `cueDone`을 영구 저장할지(누적), 세션마다 리셋할지 결정 필요(§10).

## 10. 기술 스택 & 배포 (다음 단계 구현 가이드)

- **형태:** 정적 PWA. 프레임워크는 가벼운 것 권장(바닐라 JS 또는 Preact/Svelte 등 빌드리스 우선). 단일 또는 소수 파일.
- **PWA 필수:** `manifest.webmanifest`(이름·아이콘·`display: standalone`·테마색 `#0E0F12`) + **Service Worker**(오프라인 캐시 — 헬스장/필드 무신호 대응).
- **저장:** IndexedDB(라이브러리는 `idb` 정도). localStorage는 보조(설정값).
- **백업:** 설정 화면에서 전체 데이터 JSON `Blob` 다운로드 / 파일 선택 import.
- **배포:** GitHub Pages.
  - 정적 호스팅이므로 루트에 빌드 산출물 배치(또는 `/docs` 또는 `gh-pages` 브랜치).
  - SW 등록 경로·`start_url`을 리포 서브패스(`/<repo>/`)에 맞춰 설정.
  - 가능하면 GitHub Actions로 빌드·배포 자동화.
- **아이콘/스플래시:** 192·512 PNG 필요(생성 필요).
