# CLAUDE.md — PT-GOLF (PT노트) 프로젝트 메모리

> 이 파일은 Claude Code가 리포를 열 때 자동으로 읽는 프로젝트 컨텍스트다.
> 상세 배경·시드 콘텐츠는 `docs/HANDOFF.md`에 있다. 이 파일은 요약 + 작업 규칙이다.

## 프로젝트 한 줄 요약

PT/골프 레슨 코칭 노트를 운동 중(헬스장·필드)에 한 손으로 보고 메모하는 **개인용 설치형 PWA**.
라이브: https://peoplenhpark.github.io/PT-GOLF/ · 배포: GitHub Pages (repo `peoplenhpark/PT-GOLF`).

## ⚠️ 시작 전 필수 (첫 작업 순서)

1. **리포의 실제 소스부터 읽어라.** 이 프로젝트는 이미 배포된 앱(`index.html` 등 클라이언트 렌더링 JS)이 있고, 아래 "관찰된 기능"에 없는 부분이 있을 수 있다. 추측하지 말고 실제 코드에서 데이터 모델·저장 방식·파일 구조를 먼저 확인할 것.
2. 데이터가 어디에 저장되는지 확인(로컬 `localStorage`/`IndexedDB` + 코드 내 시드 데이터 여부).
3. 그 실제 구조에 맞춰서 아래 "다음 작업"을 진행.

## 관찰된 앱 기능 (2026-07-01, 라이브 화면 기준 — 코드로 재확인 필요)

- 파트 전환: **🏋️ PT / ⛳ 골프**
- 동작 추가 폼: `부위/카테고리`, `동작 이름`, `스펙/한 줄 핵심(무게·그립·목표)`, `자세 체크리스트(줄단위)`, `잊지 말 것(줄단위, 선택)`
- 동작 삭제
- 일정 기능: `예약일 / 실시일 / 휴무일` + 시간(05~23시) 선택 캘린더
- 다크 테마(theme-color `#0d0f14`), 모바일 PWA 메타(apple-mobile-web-app-capable 등)

## 제품 방향 (확정 결정)

- 톤: **다크·미니멀 (헬스장 친화)**, 큼직한 탭/버튼, 한 손 조작.
- 데이터: **로컬 우선 + JSON export/import 백업**. 서버·로그인 없음.
- PT는 **세션(일일수행) 단위**로 묶는 구조를 지향 — 레슨이 늘면 세션(서브탭)이 추가됨.
- 액센트 색: PT = teal `#16A085`, 골프 = amber `#E0982A`. 앱 배경 `#0E0F12`, 카드 `#16181C`.
- 자세한 데이터 모델·화면 구조·디자인 토큰: `docs/HANDOFF.md` §7~§9 참조.

## 다음 작업 (우선순위)

1. **일일수행2(하체 재활) 콘텐츠 반영** — 좌골 신경통 재활 4동작(쿼드셋·SLR·클램쉘·SSLR). 입력 내용은 `docs/일일수행2_동작입력.md`, 참고 그림은 `docs/images/`. 앱 데이터 구조에 맞춰 시드/데이터 파일에 추가하거나 폼 스키마에 매핑.
2. PT **세션(일일수행) 서브탭** 개념이 현재 코드에 있는지 확인 후, 없으면 `부위/카테고리`를 세션 그룹으로 활용하거나 세션 계층 도입.
3. 동작 상세에 **참고 이미지(그림/영상 링크)** 필드 연결 — 재활 동작은 `docs/images/0X_*.png` 활용.
4. `docs/HANDOFF.md` §12 미해결 질문 확인(큐 체크 누적 여부, 편집/삭제 범위, 배포 서브패스 정합성 등).

## 코딩·콘텐츠 규칙

- UI 텍스트는 **한국어**. 재활 동작은 모두 **좌우 각각** 수행 기준.
- GitHub Pages **프로젝트 사이트**이므로 경로는 서브패스(`/PT-GOLF/`) 기준 — 절대경로 링크·SW `start_url`·manifest `scope`가 서브패스와 맞는지 항상 확인.
- 이미지는 상대경로로 참조(리포 내 위치 유지 시 GitHub·Pages 모두 렌더링).
- 변경 후에는 로컬에서 정적 서버로 띄워 확인(예: `python3 -m http.server`) 하고, PWA 설치/오프라인/백업 동작을 테스트.

## 배포

정적 사이트 → GitHub Pages. 커밋·푸시하면 Pages가 자동 반영. 배포 브랜치/폴더 설정은 리포 Settings > Pages에서 확인.
